# Changelist

| Функциональный модуль | Описание реализованного функционала | Степень завершенности | Степень проработанности | Упрощения и допущения |
| :--- | :--- | :--- | :--- | :--- |
| **Async Core (EventLoop)** | Базовый `epoll` цикл событий для обработки UDP-сокетов (RADIUS, S2a, DHCP). | 90% | Средняя. | Однопоточная архитектура. |
| **TWAN RADIUS Frontend** | UDP-сервер на порту 1812. Парсинг RADIUS заголовков и извлечение `Calling-Station-Id`. Извлечение IMSI через простой парсинг строк NAI (поиск префиксов 3GPP и удаление realm). Проксирование Accounting-Request. | 60% | Базовая. Извлечение параметров работает, но криптографическая обработка EAP-AKA отсутствует. | Игнорирование проверки `Message-Authenticator`. Аутентификация пробрасывается как есть, сложный Multi-Round стейт-машина EAP реализована поверхностно. |
| **STa Interface (Diameter)** | Интеграция с `freeDiameter`. Кодирование Diameter EAP-Request (DER) и Session-Termination-Request (STR). Асинхронная обработка Diameter EAP-Answer (DEA) и Abort-Session-Request (ASR) через статические коллбэки. Извлечение APN, PDN-Type, AMBR и IP-адреса P-GW из AVP. | 80% | Ручной обход дерева AVP (APN-Configuration, MIP6-Agent-Info) работает корректно. | Разрешение P-GW (S-NAPTR) извлечено из профиля AVP `MIP-Home-Agent-Address` напрямую (без полноценных DNS-запросов `libresolv` в самом коде). |
| **S2a Control Plane (GTPv2-C)** | Формирование `Create Session Request` и `Delete Session Request`. Упаковка IMSI в BCD, генерация локального F-TEID, привязка APN и PDN-Type. Парсинг ответов для извлечения P-GW TEID, IP-адреса абонента (PAA) и QCI. Отправка `Delete Bearer Response`. | 85% | Средняя. Используется кастомная сериализация поверх библиотеки `libgtpv2c`. Обработка базового успешного сценария. | Отсутствует обработка таймаутов, повторных передач (retransmissions) и коллизий TEID. Линейный flow. |
| **S2a User Plane (GTP-U) & QoS** | Создание устройства `gtp0` в ядре Linux через Netlink (`libgtpnl`) и настройка туннелей (локальный и удаленный TEID). Маршрутизация трафика абонента и маркировка DSCP (QoS) на основе QCI. | 80% | Средняя. Управление через Netlink работает штатно. | Маршрутизация (`ip route`) и QoS (`iptables -t mangle`) вызываются небезопасно через `system()`. |
| **Access Tunneling & DHCP** | Сервер DHCP/BOOTP (порт 67), слушающий интерфейс `twag_gre`. Отправка DHCP Offer/Ack с IP-адресом от P-GW. | 70% | Базовая. Ручная сборка пакетов DHCP, выдача статических опций (Router, DNS). | Только IPv4. Привязка сокета через `SO_BINDTODEVICE` (требует root). Нет реализации создания самого туннеля EoGRE в коде (предполагается настройка скриптами окружения). |
| **Session Lifecycle & HA** | Очистка сессий по RADIUS Accounting Stop, генерация GTPv2 Delete Session. | 50% | Базовая. | High Availability (Session Recovery), Congestion Control, Offline Charging **не реализованы** в коде. |
